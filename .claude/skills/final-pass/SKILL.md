# Final Pass Skill

Use this skill for the last quality pass before merge, deploy, or release.

## When to use

Trigger this skill when the task involves any of the following:
- final review of a page, landing page, blog page, or marketing page
- SEO verification before release
- checking whether canonical URLs, `sitemap.xml`, and `robots.txt` work correctly
- validating OG image coverage and generation
- validating mobile layout and responsive behavior
- confirming fonts are local and no external Google Fonts are loaded
- running or fixing `@casoon/astro-post-audit`
- updating or verifying Playwright SEO/mobile coverage

## Primary goals

1. Verify SEO completeness
2. Verify canonical, sitemap, and robots behavior
3. Verify OG image coverage
4. Verify mobile quality
5. Verify self-hosted fonts only
6. Verify build-time and E2E quality gates
7. Verify performance basics (render-blocking, image optimization, lazy loading)

## Required checks

### SEO
- page title is correct
- meta description exists and is sensible
- Open Graph tags exist and use correct values
- Twitter card tags exist where expected
- JSON-LD is present and valid for the page type
- `noindex` is used where appropriate, such as 404 pages or excluded pages

Use the `seo` skill for project-specific implementation details.

### Canonical
- every indexable page has exactly one correct canonical URL
- canonical points to the final public URL
- canonical matches locale, domain, and trailing-slash behavior
- canonical is consistent with the sitemap entry
- 404 and noindex pages do not expose misleading indexable canonicals

This is a hard requirement for final review.

### Sitemap and robots
- `sitemap.xml` responds successfully
- sitemap should be generated by project code, not by an external sitemap plugin
- prefer `createSitemapRoute()` or an equivalent local route helper; do not add `@astrojs/sitemap` for this workflow
- sitemap contains the expected important URLs
- sitemap excludes drafts and known redirect artifacts
- `robots.txt` responds successfully
- `robots.txt` references the correct sitemap URL

This is a hard requirement for final review.

### OG images
- each important page has a valid `og:image`
- OG image URLs are absolute and publicly correct
- generated OG assets exist for both locales where required
- blog post OG images are generated from content where expected
- OG image generation scripts still work when page coverage changes

### Mobile
- verify layout on small viewport sizes
- no horizontal overflow
- navigation works on mobile
- headings, spacing, and CTA blocks remain readable and usable
- forms, cards, and interactive elements remain usable on touch devices
- no obvious clipping, overlap, or broken stacking

Use Playwright where practical and prefer evidence over assumption.

### Fonts
- fonts must be self-hosted
- prefer `@fontsource` or local assets already used by the repo
- no dependency on `fonts.googleapis.com` or `fonts.gstatic.com`
- preserve current project font patterns unless the task explicitly changes typography

Use the existing project font setup as the default pattern.

### Performance

Check for common PageSpeed issues that are detectable from code:

#### Images
- LCP image (first visible hero/teaser image) has `fetchpriority="high"`
- LCP image does NOT have `loading="lazy"`
- `sizes` attribute reflects actual display dimensions, not viewport width
- Below-the-fold images have `loading="lazy"`
- No oversized images served (e.g., 756w for a 380px slot)

#### CSS
- No duplicate `@font-face` declarations across imported stylesheets
- No duplicate utility class definitions across CSS files
- Font files use `font-display: swap`
- Consider font subsetting when font files exceed 50 KiB

#### JavaScript and data
- Large data files (search indices, JSON) are lazy-loaded on interaction, not on page load
- No unnecessary scripts in the critical path
- Third-party scripts (analytics, tracking) loaded via server-side injection (e.g., Zaraz) or deferred

#### Resource hints
- `<link rel="preconnect">` for critical third-party origins
- No preloading of resources that are not used on the current page

This is a recommended check, not a hard blocker. Report findings with estimated impact.

### Post-audit
- run the build path that triggers `@casoon/astro-post-audit`
- treat audit findings as real issues unless clearly identified as false positives
- fix root causes instead of suppressing by default
- only add exclusions when the output is intentionally non-content, such as redirect artifacts

Use the `post-audit` skill for integration details.

### Playwright
- run or update existing SEO and relevant responsive checks
- prefer extending existing `seo.spec.ts` coverage over adding scattered new tests
- when mobile behavior changed, add or update targeted checks instead of hand-waving

Use the `playwright` skill for project test conventions.

## Workflow

1. Inspect the changed page or feature
2. Review SEO implementation
3. Verify canonical behavior
4. Verify `sitemap.xml` and `robots.txt`
5. Prefer a custom sitemap route or shared utility over a sitemap plugin
6. Verify OG image coverage and generation
7. Verify fonts are local
8. Check performance basics (images, CSS, lazy loading)
9. Build the relevant app and review post-audit findings
10. Run or update Playwright checks
11. Review mobile behavior
12. Report blockers first, then medium-risk issues, then optional improvements

## Output expectations

When using this skill:
- findings come first
- include concrete file references
- clearly separate blockers from polish
- if all checks pass, say so explicitly
- mention what was validated and what was not run
