Tailwind v4 introduces a fundamental shift: **CSS-first configuration** instead of JavaScript.

## What Changed?

- No more `tailwind.config.js` needed
- Configuration directly in CSS via `@theme`
- Native OKLCH colors
- Built-in container queries
- Significantly faster builds

## Integration with Astro v6

We use the Vite plugin directly:

```javascript
import tailwindcss from '@tailwindcss/vite';

export default defineConfig({
  vite: {
    plugins: [tailwindcss()],
  },
});
```

## Design Tokens

Our design tokens use CSS custom properties with OKLCH colors for consistent, accessible color schemes in both light and dark mode.
