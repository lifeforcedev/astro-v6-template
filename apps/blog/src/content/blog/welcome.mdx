This is the first post in the **Astro v6 Blog Template**. Here we cover the most important changes.

## What's New in Astro v6?

- **New dev server** powered by the Vite Environment API
- **Live Content Collections** for real-time data
- **Content Security Policy** out of the box
- **Node 22+** as minimum requirement
- **Zod v4** for schema validation
- **Vite 7** under the hood

## Content Collections with Loader API

In Astro v6, Content Collections use the new Loader API. Instead of `type: 'content'`, we now define a `loader`:

```typescript
import { defineCollection } from 'astro:content';
import { glob } from 'astro/loaders';
import { z } from 'astro/zod';

const blog = defineCollection({
  loader: glob({ pattern: '**/*.mdx', base: './src/content/blog' }),
  schema: z.object({
    title: z.string(),
    date: z.coerce.date(),
  }),
});
```

Astro v6 brings significant improvements under the hood that bring the development workflow closer to production.
